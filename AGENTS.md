# LLM Instructions

## Code Standards

ALWAYS apply KISS and DRY principles
ALWAYS write minimal code
ALWAYS follow PEP 8 conventions
ALWAYS use type hints for all functions
ALWAYS prioritize readability over performance unless explicitly requested
ALWAYS comment to explain WHY, not WHAT
Always be sure that fonction has 50 code ligne maximum

## Documentation Rules

ALWAYS write docstrings in English with consistent format
ALWAYS add inline comments to justify design choices

## Architecture Rules

ALWAYS place reusable functions in src/utils.py
ALWAYS create only the single function explicitly requested
NEVER create multiple functions at once

## Testing Rules

ALWAYS create unit test immediately after creating any function
ALWAYS use mocked data for tests
NEVER use real data in unit tests

## Interaction Rules

NEVER assume methodology or requirements
ALWAYS ask for clarification when anything is unclear
